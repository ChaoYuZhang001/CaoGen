# Security Policy

CaoGen 是本地桌面 AI 编码工作台，会接触项目源码、终端输出、模型 Provider Key、Git 仓库和本机文件系统。安全问题请按本文件处理。

## 支持版本

| 版本 | 安全支持状态 |
|---|---|
| `main` | 接收安全修复 |
| 最新 Release | 接收高优先级安全修复 |
| 旧版 beta | 尽力支持，建议升级到最新版本 |

## 报告漏洞

请不要在公开 Issue、PR、讨论区或截图中直接披露可利用细节、密钥、访问令牌、私有仓库路径或完整日志。

优先使用 GitHub 的 private vulnerability reporting 或 Security Advisory。如果仓库暂未开启私密报告入口，请创建一个不含漏洞细节的公开 Issue，标题使用 `[security]`，正文只说明你需要安全联系方式，维护者会跟进。

报告中建议包含：

- 受影响版本或 commit。
- 操作系统和安装方式。
- 漏洞类型和影响范围。
- 最小复现步骤。
- 是否已经出现密钥、源码、文件或远程服务暴露。
- 你建议的修复方向，如果有。

## 密钥泄露处理

如果 API Key、GitHub token、模型 Provider Key 或其他凭据已经出现在公开位置：

1. 立即在对应平台撤销或轮换密钥。
2. 检查相关账号、组织和仓库的访问日志。
3. 从仓库历史、Release、Issue、PR、构建日志和本地缓存中移除可见副本。
4. 再按上方流程提交安全报告。

只删除文本通常不够，已经公开过的凭据应视为失效。

## 本地安全边界

- CaoGen 没有自己的云端代码托管服务。
- 你发给 Agent 的上下文、文件片段、命令结果和工具输出会发送给你选择的模型 Provider、网关或本地模型服务。
- 命令执行、文件写入、GUI 自动化、worktree 自动合并和 PR/MR 创建都属于高风险能力，应在可信项目中使用。
- macOS v0.1.5 安装包未签名、未公证，首次打开需要用户手动确认。

## 维护者响应

安全报告会优先确认影响范围，再决定是否临时下架 Release、撤销资产、发布修复版本或补充检测脚本。复杂问题可能需要更多复现材料，但请先避免公开披露细节。
