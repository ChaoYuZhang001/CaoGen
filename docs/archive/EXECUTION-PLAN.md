# CaoGen 执行计划：楔子发布 → 用户 → 贡献者 → 投资

> 制定日期：2026-07-22
> 性质：把"全部功能做完再上线"的必死赌注，换成"先发能用的楔子 → 拿真实用户 → 招志同道合的贡献者 → 用牵引力换投资 → 再带团队做全功能"。
> 事实基线以 [`STATUS.md`](../STATUS.md) 为准；本文件是**执行计划**，不是能力宣称。

### 0.1 三方分工

| 角色 | 职责 |
|---|---|
| **创始人（你）** | 战略决策、签名发布、创建真 issue、发布帖、投资对话。只有你能做的事见 §5。 |

### 0.2 铁律（违反即失败，优先级最高）

1. **绝不夸大。** 项目核心资产是"诚实不虚标"。所有对外文案只宣传**当前已发布/能用**的能力；立项目标必须标注为"路线图/建设中"。**不许把 42 个立项目标写成已完成。**
2. **不删愿景文档。** `PROJECT-CHARTER.md`、`PRODUCT-REQUIREMENTS.md`、`STATUS.md`、`ROADMAP.md` 全部保留，只在对外物料里清楚区分"现在能用" vs "长期愿景"。
3. **不碰签名凭据、`.env`、release 流程**——人工在本机做。
4. **中英双语，zh 母语级。**
5. **发一个诚实标注 beta 的楔子 ≠ 撒谎。** 可以骄傲地说"能用的厂商中立 AI 桌面，完整 Agent Work OS 愿景在路上"——每个字都真。

## 1. 战略重定位（为什么这么排）

### 1.1 把"1.0 = 全功能"拆成两个东西

- **CaoGen 楔子版（0.2）= 今天就能用的一切**：多 Provider + BYOK + 智能路由 + 跨厂商 failover + worktree 隔离 + 工作台（终端/浏览器/Diff/Git/Office 预览）+ 3D 办公区（机器人版，不等水墨）。
- **CaoGen Agent Work OS 愿景（2.0）= 现在整套 1.0 定义**：数字员工、Goal/WorkItem/Acceptance、canonical ledger、Native Runtime、水墨 3D。**一个字不删，只改标签为"2.0 路线图 / 长期愿景"，让它停止阻塞发布。**

### 1.2 顺序倒转

你原本的顺序 `全功能 → 用户 → 同好 → 投资` 会卡在第一步，后面三个都拿不到。真实可达的顺序是它的反面：

```
发楔子 → 拿用户 + 同时开放贡献 → 牵引力 → 投资 → 带团队做全功能
```

原因：**用户、贡献者、投资，没有一个需要"全部功能"。** 用户要一个能打的楔子；贡献者是造功能的手段不是造完的奖励；投资投的是牵引力+团队+楔子，不是功能完整度。

### 1.3 立刻暂停的事

- Effect Ledger v9、reconciler 扩展、canonical conversation ledger、水墨美术——**对 0 用户是过早优化**。把时间转到阶段 1-4。等有用户、有数据要保护，再回来做崩溃恢复。

## 3. 阶段详解

### 阶段 0 · 切楔子（2-3 天，纯决策）

全计划最重要的一步，心理上最难：把"1.0=全功能"拆成"楔子版 0.2" + "2.0 愿景"。

**动作：**
1. 写一句定位（贴 README/官网顶部）：**"不锁厂商的本地 AI 工作桌面——用你自己的 key 跑任意模型，一个挂了自动换。"**
2. 把水墨 3D、Native Runtime、canonical ledger 从"发布门禁"解绑。
3. 暂停 §1.3 列出的过早优化。

**Gate：** 你能一句话说清"现在要发的是什么"，且不依赖任何未完成的立项目标。

### 阶段 1 · 能装（第 1 周，创始人）

签名/公证管线已跑通（notarytool 已认证成功），只是被锁在"完美 1.0 干净候选"门禁后。解锁它。

**动作：**
1. 从当前能跑的 commit 拉一个**干净 tag**（只需该 commit 干净，不需整个 worktree 干净）。
2. 对它跑 `打包 → codesign → notarize → staple → 审计`（管线现成）。
3. 发签名+公证的 macOS 下载到 GitHub Release；Windows 同理。
4. 官网 `signatureState` 改掉，README 去掉"右键打开"摩擦，保留 beta 标注。

**Gate：** 陌生人下载双击，App 正常打开，无"来源不明"弹窗。

### 阶段 2 · 能懂（第 1-2 周）

**动作：**
1. **录 30-60 秒 demo**（GIF+视频）：加 3 家不同厂商 key → 一个挂了自动切 → 出结果。（创始人录，Sol 可写脚本）
2. **改 README/官网第一屏**：喊楔子，把"5/17/42 未完成"下移到中后段。（Sol，见 §4 任务 1）
3. **英文 README**。（Sol，见 §4 任务 2）

**Gate：** 发给不懂项目的朋友，10 秒说得出"这是干嘛的"并看过 demo。

### 阶段 3 · 能贡献（第 2-3 周，Sol 执行）

招同好不是有用户之后的事——这就是招同好的动作本身。详细任务见 §4。

**Gate：** 没见过项目的人能在 issue 列表找到明确任务、跑起来、提 PR，全程不需理解 Effect Ledger。

### 阶段 4 · 发布获客（第 3-5 周，创始人）

去有人的地方，用一句楔子。**按顺序发，不要一天全发。**

1. **r/LocalLLaMA + r/ChatGPTCoding**（英文）——信仰 local+BYOK+vendor-neutral，最天然受众。
2. **Hacker News**（Show HN）——开源+厂商中立+3D 角度。
3. **国内**：V2EX、掘金、少数派、即刻、Bilibili（demo 视频）。
4. **ProductHunt** 一次正式 launch。
5. 每个渠道主标题同一句：**"我做了个不锁厂商的本地 AI 桌面，用你自己的 key 跑任意模型"**。第一层只讲楔子，Agent Work OS 愿景是发布后第二层。

**Gate：** 100-500 装机 / star 起量 / 首个非本人 issue 或 PR。

### 阶段 5 · 牵引力循环（第 2-3 月）

把"用户"变成"可融资资产"。

**动作：**
1. **快修循环**：真实 bug 当周修当周发，周更节奏。
2. **功能优先级重排**：用户投票最多的先上——42 个立项目标从此按真实需求重排。
3. **埋点跟踪可融资数字**：装机量、**周留存**、star、贡献者数、活跃 issue/PR。
4. 留住第一批贡献者（这本身证明"团队能组建"）。

**Gate（3 个月目标，是靶不是承诺）：** 首批 100+ 真实装机 / 500-2000 star / 5+ 外部贡献者 / 20%+ 周留存 / 零市场预算。

### 阶段 6 · 投资叙事（第 3 月起）

**动作：**
1. 写**一页 wedge story**（不是 28KB 立项书）：
   - 楔子：厂商中立+本地+开源+国产直连（两大厂结构上做不了）
   - 牵引力：阶段 5 的数字
   - 团队能力：活跃贡献者 = 能组建团队
   - 市场：信创/国产化/数据不出境——2026 有钱看的主题
   - 模式：open-core（AGPL+商业授权，参照 GitLab/Dify）
2. 找对的钱：国内 AI 应用天使/早期基金 + 对开源友好的机构。参照 **Dify**（中国人做的开源 AI，靠用户+star 拿到融资）。
3. 带牵引力谈，不是带功能清单谈。

**Gate：** 进入投资人对话，筹码是牵引力不是"我快做完了"。

## 4. ChatGPT-Sol 仓库改造执行简报

> Sol 无本对话上下文，本节自包含。执行前先读 §0.2 铁律。目标：把 GitHub 仓库改造成能吸引开源贡献者的样子，核心是重写 README。

### 任务 1 · 重写 README（最高优先）
按此结构改 `README.md`：
1. **第一屏（10 秒看懂）**：一句定位 + 顶部 demo GIF/截图（`docs/screenshot-app.jpg` / `screenshot-office.jpg`）+ 保留 badges。
2. **"这是什么"两句话** + 一张主界面图。
3. **核心能力（只列已发布/能用，诚实标注）**：多 Provider/BYOK、智能路由+跨厂商 failover、worktree 隔离、工作台（终端/浏览器/Diff/Git/Office 预览）、3D 办公区。每条一行，动词开头。
4. **Quick Start**：下载 → 加 Provider/key → 第一个任务，3 步。
5. **长期愿景（清晰隔离）**：一段 + 链接 Charter，标题"Roadmap / 长期愿景（建设中）"，不与当前能力混淆。
6. **贡献区（显眼）**：链接 CONTRIBUTING、good-first-issues、"我们在找志同道合的人"。
7. 顶部加 `English | 简体中文` 切换。

**验收：** 陌生开发者扫一眼能说出"这是干嘛、和 ChatGPT/Claude 桌面版差别、怎么装、怎么参与"。

### 任务 2 · 英文 README
生成 `README.en.md`，两文件顶部互链。英文是全球贡献者入口，招人杠杆最高。同守铁律。

### 任务 3 · 仓库元数据
- 更新 repo **Description**（一句楔子）。
- 加 **Topics**：`ai-agent` `llm` `electron` `byok` `vendor-neutral` `local-first` `multi-provider` `deepseek` `claude` `openai` `agent-desktop` `react-three-fiber`。
- 建议一张 **social preview 图**放 `docs/`，PR 描述里提示人工上传。

### 任务 4 · CONTRIBUTING + 架构地图
写 `CONTRIBUTING.md`（中英）：怎么跑起来（`npm install`/`npm run dev`）；**架构地图**（主进程→IPC→preload→types→store→UI 六环链路 + `src/` 目录说明）；从哪开始（指向 good-first-issues）；PR 流程 + 承诺 48 小时内回应。

### 任务 5 · 起草 10 个 good-first-issue
**都必须：不碰核心账本（Effect/Workflow Ledger）、外人能独立做完。** 先用 `wc -l`/grep 核对代码再定稿，输出到 `docs/good-first-issues.md`（人工再建真 issue）。方向：
1. 新增一个国产 Provider 模板（参考现有模板结构）
2. 文件预览新增一种格式（`.yaml`/`.toml`/`.xml`）
3. i18n zh/en 缺失字符串补齐 + 自动 parity 检查脚本
4. 工作台某按钮补 tooltip/快捷键/aria-label
5. 给某纯函数 helper 加 smoke 测试
6. 修一处错误文案（不清晰/中英不一致）
7. Monaco 编辑器补一种语法高亮语言
8. 工具结果卡片加"复制到剪贴板"按钮
9. 某网关类型加连通性检测
10. 给 `scripts/` 里某 smoke 脚本写文档

### 任务 6 · Issue/PR 模板
`.github/ISSUE_TEMPLATE/`（bug/feature/good-first）+ `PULL_REQUEST_TEMPLATE.md`。

### 任务 7 · 拆巨型文件（较大，放最后）
先 `wc -l src/main/*.ts src/renderer/src/**/*.tsx | sort -rn | head` 找最大的 3 个，**只拆最大 1-2 个**，按职责切模块、行为不变、`npm run typecheck && npm run build` 验证。降低贡献门槛用，别一次性大重构。

### Sol 每任务完成后

## 7. 两条贯穿纪律（再强调）

1. **诚实照旧，但"不完美就不发"要改成"诚实地发不完美"。**
2. **全功能不是被放弃，是被推迟到"有人有钱"之后**——账本、数字员工、水墨 3D 都会做，带着团队和融资做，不是一个人不拿钱先做完。

> **一句话：功能是终点，不是起点。现在要做的是把一小片楔子推出门、让世界看到它活着。**
