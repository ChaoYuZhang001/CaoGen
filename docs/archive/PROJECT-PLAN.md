# CaoGen 项目计划（Master Plan）

> 制定日期：2026-07-22 · 这是**顶层项目计划**，用来跑项目和追踪进度。

> 事实基线以 [`STATUS.md`](../STATUS.md) 为准。

## 1. 一页纸概览

| 项 | 内容 |
|---|---|
| **产品** | 不锁厂商的本地 AI 工作桌面（Electron + React，AGPL）；长期愿景是 Agent Work OS |
| **一句定位** | 用你自己的 key 跑任意厂商模型，一个挂了自动换，不被任何厂商锁定 |
| **当前状态** | v0.1.6 未签名已发布；楔子功能可用；签名管线就绪但被 1.0 门禁锁住；0 外部用户 |
| **战略** | 先发能用的签名楔子 → 拿真实用户 → 招贡献者 → 用牵引力换投资 → 再带团队做全功能 |
| **下一个里程碑** | **M1：发出第一个签名版（0.2.0 macOS）** |
| **最大风险** | 0 外部用户、从未真人验证；以及"不完美就不发"锁死发布 |

## 2. 目标与成功定义

**终极目标（你的）：** 全部功能上线 → 用户用 → 志同道合之人一起优化 → 投资商业化。

**达成路径（把顺序倒过来）：** 功能是终点不是起点；用户/贡献者/投资都不需要"全功能"，需要一个能打的楔子 + 分发 + 开放。

**成功指标：**

| 层级 | 指标 | 近期靶（3 个月，非承诺） |
|---|---|---|
| 激活 | 真实装机量 | 100-500 |
| 认可 | GitHub star | 500-2000 |
| 团队 | 外部贡献者 | 5+ |
| 留存 | 周留存 | 20%+ |
| 融资 | 牵引力驱动的投资对话 | 启动 |

**北极星（0 用户可动版）：** 完成一次真实任务、下周还回来的真实用户数。

## 7. 风险与铁律

**铁律：**
1. 绝不夸大——只宣传已发布能力，立项目标标"建设中"。
2. 签名 ≠ 宣称 stable；诚实标 beta。
3. 干净 commit ≠ 干净 worktree；正式包只从 tag 出。
4. Required 永不 skip 充数；发布物不含凭据。
5. 全功能不是放弃，是推迟到"有人有钱"之后。

**风险：**
- 0 用户 / 从未真人验证 → M4 前插入真人试用，别再拖到"完美"。
- "不完美就不发"锁死发布 → 靠 Beta 门禁解绑。
- Sol 为吸引人把 README 吹大 → 合并前你+Claude 双查。
