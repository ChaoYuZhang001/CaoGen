## 1. Purpose And Closure Rule

This matrix converts the CaoGen 1.0 product baseline into executable release work. It covers all 64 P0 requirements in `PRODUCT-REQUIREMENTS.md`, all 38 P1 requirements for selection and Go review, and the additional human, time, platform, signing, packaging, and publication gates in `PROJECT-CHARTER.md`.

A row is closed only when all of the following are true:

1. The implementation exists across every required main/IPC/preload/shared/store/UI boundary.
2. The listed automated or human gate passes against the exact release commit.
3. The evidence records the commit, version, platform, environment, and required/optional state.
4. The current limitation and external conditions are reflected in `STATUS.md` and release copy.
5. No open dependency in the final column invalidates the result.

`Current status` is copied from the PRD baseline. It is not upgraded by this matrix. Commands prefixed with **Add gate** do not exist yet; implementing that required gate is part of closing the row. Later rows may reference those proposed gates without repeating the prefix, but the reference does not turn them into existing commands.

## 2. Gate Classes

| `HUMAN-TIME` | Requires a human acceptance run, timed drill, rights review, or elapsed soak period. |

## 8. Parallel Execution Order

1. `L0-GATE` adds the machine-readable requirement map and makes stale/unbound evidence fail closed.
2. `L2-PROJECT` and `L5-TRUST` freeze Project/Goal/WorkItem/Workflow/Artifact ownership and canonical event contracts.
3. `L3-WORKER`, `L4-ROUTING`, `L6-ARTIFACT`, and `L7-AUTOMATION` implement against those contracts in parallel.
4. `L1-EXPERIENCE` and `L8-VISUAL` can prototype concurrently, but their final E2E must consume the real canonical objects from steps 2-3.
5. `L9-RELEASE` maintains performance and packaging harnesses continuously; clean Deep, signing, notarization, final packaging, and publication run only after product freeze.
6. Human workflow and N1 gates use the feature-frozen candidate and cannot be replaced by mocks, screenshots, or internal developer self-assertion. Seven-day soak follows the same rule by default; exact `1.0.0` instead carries the explicit owner waiver and accepted residual risk in `docs/1.0-SOAK-WAIVER.json`.

## 9. Final Go Record

The final Go record must contain:

- exact commit, tag, package/lockfile version, branch, and clean-worktree proof;
- 64/64 P0 closure and the frozen P1 disposition list;
- every automated report path and its commit/environment binding;
- every human/time/platform record and reviewer;
- signed/notarized artifact names, sizes, SHA-256, architecture, and artifact-set digest;
- accepted residual risks and requirement-specific waivers;
- final Release Notes body and post-publication asset/read-text audit.

Until that record exists, `1.0.0` is a local candidate version, not proof that CaoGen 1.0 stable has met its product completion definition.
