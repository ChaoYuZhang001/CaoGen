# CaoGen 1.0 Seven-Day Soak

Run the soak only after the 1.0 candidate is feature-frozen. Any source, dependency, configuration-default, or packaged-artifact change resets the seven-day clock.

The default release policy still requires this evidence for 1.x stable releases. CaoGen `1.0.0` has one explicit exception: the release owner accepted the residual risk in `docs/1.0-SOAK-WAIVER.json`. The audit reports that decision as `waived`, never as `passed`, and Release Doctor makes `product_1_0_soak` non-blocking only when the requested version is exactly `1.0.0` and every waiver field validates. The waiver expires with `1.0.0` and cannot be inherited by `1.0.1`, another 1.x release, or a renamed target.

1. Copy `docs/1.0-SOAK-RESULT.template.json` to a private path outside the repository.
2. Record at least one real daily workflow on seven consecutive calendar dates.
3. Copy the exact artifact-set SHA-256 from the release packaging audit and keep the candidate version, Git commit, and artifact-set digest identical on every day.
4. Record data loss, duplicate high-risk effects, blocking defects, and every defect disposition truthfully.
5. Keep raw logs, recordings, provider identifiers, user names, secrets, and private project paths in the private evidence root.
6. Validate the private record with `npm run test:1.0-soak-audit:required -- --record /private/path/soak.json`.
7. Run Release Doctor from the same clean candidate commit. A passed record for another version or commit is rejected.

The audit validates evidence structure and release identity. It cannot create seven elapsed days or replace human daily use.

## 1.0.0 Owner Waiver

Validate the tracked decision with:

```sh
npm run test:1.0-soak-audit:required -- --version 1.0.0 --waiver docs/1.0-SOAK-WAIVER.json
npm run workos:release-doctor -- --version 1.0.0
```

The waiver records the gate ID, exact version, owner, decision date/source, reason, accepted risk, expiry, exclusion of future versions, and compensating evidence. It waives elapsed time only. Formal P0 acceptance, clean Deep, real-provider evidence, SBOM, secret history, signing, notarization, stapling, packaging, final notes, and every other Release Doctor gate remain unchanged and blocking.
