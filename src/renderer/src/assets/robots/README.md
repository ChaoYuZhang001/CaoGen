# Reference office robot

`reference-office-robot.glb` and `reference-office-robot-lod.glb` are generated by
`scripts/generate-reference-robot-glb.mjs` from the official Unitree G1 rev 1.0
assets vendored in `third_party/unitree-g1-rev1`. Both assets use Draco mesh
compression. The low-detail asset is rebuilt from the same hierarchy by applying
a deterministic decimation pass after the full-detail Blender source has been
saved, so the editable source never becomes the decimated model.

The visible service helmet is maintained separately in
`reference-helmet-source.blend`. It contains the editable, single-island quad
shell and its visor/sensor details; production generation appends that source
collection to the pinned Unitree body instead of rebuilding the helmet from
transient primitives.

Regenerate assets in dependency order:

```sh
npm run generate:reference-helmet-source
npm run generate:reference-robot
```

Then run `npm run test:reference-robot-glb` for the full asset and pass the LOD
asset path to the same smoke for its hierarchy, topology, provenance, and
animation-root contracts:

```sh
node scripts/reference-robot-glb-smoke.mjs \
  src/renderer/src/assets/robots/reference-office-robot-lod.glb
```

`npm run test:reference-robot-silhouette` compares
the checked-in front/side renders with the orthographic reference and enforces a
regression baseline. That gate prevents the current silhouette from getting
worse; it does not claim pixel-level parity with the reference artwork.

The renderer decodes both files with the Draco WebAssembly decoder copied from
the pinned `three` dependency into `src/renderer/public/draco`; no CDN is used.

The GLB contract follows exact official-source mesh names rather than the old
hand-built palm-shell and helmet-part names:

- `official_head_link` binds the `head_link` mesh.
- `official_torso_link` or `official_torso_link_23dof_rev_1_0` binds the
  `torso_link_23dof_rev_1_0` mesh.
- The current export uses `official_left_rubber_hand` and
  `official_right_rubber_hand`. A full 23-DOF export may use the exact upstream
  names `official_left_wrist_roll_rubber_hand` and
  `official_right_wrist_roll_rubber_hand` instead.
- Runtime animation hooks bind the exact roots `helmet_head`, `left_arm`,
  `right_arm`, `left_leg`, and `right_leg`.

The official `logo_link` mesh is not baked into the office GLB. Provider/vendor
logos are rendered at runtime by `ProviderLogoBadge` at
`provider_nameplate_mount`.
